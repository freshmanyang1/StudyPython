# 交接协议

角色之间的交接只保留 5 类模板：完整交接、轻量交接、测试报告、审查报告、BLOCKED。任务状态流转规则见 `ai/TASK_FLOW_RULES.md`。

## 最小字段

除项目经理“立任务”外，任何角色完成一次处理必须写清：

```markdown
### 本次完成

1. 做了什么
2. 改了什么
3. 验证了什么 / 没验证什么

### 状态变更

旧状态 -> 新状态

### 主执行 Agent

codex / cursor / claude / 其他：名称

### 下一步处理身份

项目经理 / 测试 / 审查 / 架构师 / 后端工程师 / 前端工程师 / aiskill / 无

> **防呆警告**：绝对禁止将 `下一步处理身份` 填写为 `flow`。控制权交还 flow 是动作，但身份必须填写具体的业务角色。

### 下一步动作

一句话说明下一身份要做什么。

### 验证/未验证

- 已验证：
- 未验证：
```

`下一步处理身份` 缺失时默认交项目经理补充或裁决；若任务状态、Owner、锁、PM 决策和任务说明均无冲突，flow 允许按状态归属兜底一次并补写交接。与状态冲突时仍必须交项目经理。

## 完整交接

用于完整任务、跨文件实现、返工或项目经理收口。

````markdown
## 任务交接 [YYYY-MM-DD]

**任务 ID**：T-XXX-NNN
**Owner**：角色名
**主执行 Agent**：codex / cursor / claude / 其他：名称
**状态变更**：旧状态 -> 新状态
**下一步处理身份**：角色名 / 无
**下一步动作**：一句话说明。

### 本次完成

1. 做了什么。
2. 改了什么。
3. 验证了什么 / 没验证什么。

### 修改文件

- `路径`：说明改了什么。

### 验证/未验证

- 已验证：
- 未验证：

### 风险

- 无 / 具体说明。
````

## 轻量交接

用于临时修复、aiskill 协作资产任务、纯规则同步或状态字段修复。

````markdown
## 轻量交接 [YYYY-MM-DD]

**任务 ID**：T-XXX-NNN
**Owner**：角色名
**主执行 Agent**：codex / cursor / claude / 其他：名称
**状态变更**：旧状态 -> 新状态
**下一步处理身份**：角色名 / 无
**下一步动作**：一句话说明。

### 本次完成

1. 做了什么。
2. 改了什么。
3. 验证了什么 / 没验证什么。

### 自检

- 范围检查：通过 / 未覆盖，原因：
- 行尾检查：通过 / 未覆盖，原因：
- 未运行命令：说明原因。
````

## 测试报告

````markdown
## 测试报告 [YYYY-MM-DD]

**任务 ID**：T-XXX-NNN
**主执行 Agent**：codex / cursor / claude / 其他：名称
**测试结论**：通过 / 不通过
**状态变更**：CODE_READY -> REVIEW_READY / CODE_READY -> TEST_FAILED
**下一步处理身份**：审查 / 项目经理
**下一步动作**：一句话说明。

### 覆盖范围

1. 测试了哪些范围。
2. 覆盖了哪些验收点。
3. 验证了什么 / 没验证什么。

### 问题分类

- 阻断可用：无 / 具体说明
- 验收缺口：无 / 具体说明
- 验证环境问题：无 / 具体说明
- 建议补测：无 / 具体说明
````

## 审查报告

````markdown
## 审查报告 [YYYY-MM-DD]

**任务 ID**：T-XXX-NNN
**主执行 Agent**：codex / cursor / claude / 其他：名称
**审查结论**：未发现对应级别问题 / 发现问题清单
**有效代码行**：数字 / 0 / 无法统计：原因
**建议动作**：原任务 DONE / 原任务 DONE + 项目经理承接 / 原任务 DONE + 阻断承接或 PM 决策
**承接要求**：无 / 需要项目经理创建阻断承接任务 / 需要 PM 决策 / 已有承接任务 T-XXX-NNN
**PM 决策 ID**：无 / D-YYYYMMDD-NNN
**下游暂停**：否 / 是，暂停任务：T-XXX-NNN
**下一步处理身份**：项目经理 / 无
**下一步动作**：一句话说明。

### 问题清单

- bug：无 / 具体说明
- 风险：无 / 具体说明
- 优化建议：无 / 具体说明
- P0/P1/P2/P3 分级问题：无 / 具体说明

P0/P1 问题必须填写 `承接要求` 或 `PM 决策 ID` 之一，并明确 `下游暂停`；审查角色不得提前填写尚未创建的任务 ID。没有 P0/P1 时三项均可填 `无` / `否`。

### 验证/未验证

- 已验证：
- 未验证：
````

## BLOCKED

````markdown
## BLOCKED [YYYY-MM-DD]

**任务 ID**：T-XXX-NNN
**当前 Owner**：角色名
**主执行 Agent**：codex / cursor / claude / 其他：名称
**状态变更**：当前状态 -> BLOCKED
**下一步处理身份**：项目经理
**下一步动作**：评估阻塞原因并裁决补充任务、改派、拍板或关闭。

### 阻塞原因

具体说明为什么不能继续。

### 需要什么

需要谁做什么决策或补充什么任务。

### 影响范围

- 受影响模块：
- 受影响任务：
````
