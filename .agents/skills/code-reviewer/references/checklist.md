# Code Reviewer Checklist

## 代码审查问题分级（code-reviewer 专属）

- P0：会导致数据泄露、认证绕过、协议不可用、服务崩溃、设备大面积不可连接的问题。
- P1：会导致主要功能失败、三端协议不一致、资源泄露、并发错误、兼容性破坏的问题。
- P2：会导致可维护性明显下降、扩展点被锁死、错误处理不明确、局部功能不稳定的问题。
- P3：低风险问题，包括命名、局部可读性、轻微重复、建议性改进。

该分级仅用于 code-reviewer 最终审查报告；P0/P1 会触发 `ai/TASK_FLOW_RULES.md` 中的阻断承接机制。

没有真实问题时，明确输出“未发现 P0-P3 级别问题”，并说明剩余风险或未覆盖范围。

## 通用必查清单

### 1. 范围与契约

- 是否越过任务允许范围，修改了禁区文件、无关模块或未授权仓库。
- 是否修改公共 DTO、接口、枚举、协议字段、事件名、Topic、缓存 key、数据库字段或外部 API，却没有确认唯一事实来源和兼容策略。
- 是否需要同步更新生成代码、前端类型、SDK、文档、迁移脚本、测试夹具或调用方契约。
- 是否把临时适配写成长期公共接口，导致后续扩展被当前实现锁死。

### 2. 数据语义与副作用

- 是否混淆业务实体、缓存索引、连接状态、日志记录、统计快照和外部通知的语义边界。
- 写库、写缓存、发消息、调用外部服务、写文件或推送通知失败时，接口成功语义是否清楚；是否需要重试、回滚、补偿或可观测告警。
- 删除、更新、批量操作是否遗漏关联数据、反向索引、二级缓存、幂等键或历史兼容路径。
- 默认值、自动选中、默认开启、默认数量上限、过期时间和重试次数是否有业务依据，并说明变更影响。

### 3. 异步、并发与资源生命周期

- 是否存在 `.Result`、`.Wait()`、阻塞式异步调用、`async void` 或不可观测的 fire-and-forget。
- 后台任务是否捕获请求作用域服务、吞掉异常，或在应用退出时丢失关键工作。
- 共享状态、连接映射、会话替换、重复释放、竞态更新、批量并发是否有锁、幂等或一致性设计。
- 网络连接、文件流、数据库连接、订阅、定时器、CancellationToken、Stream、WebSocket/TCP 等资源是否正确释放和取消。

### 4. 错误可见性与安全

- 是否吞异常、只打日志后返回默认值、抛出过宽异常，或日志缺少定位问题所需的关键上下文。
- 是否在日志、异常、响应、前端状态或测试输出中泄露 token、密钥、连接串、敏感 header、访问 URI、个人信息或业务敏感数据。
- 权限、认证、授权、租户隔离、输入校验、反序列化、路径访问和上传下载边界是否被绕过。
- 错误处理是否区分可降级、需重试、需阻断和需用户修正的场景。

### 5. 兼容性与用户意图

- 是否破坏旧客户端、旧数据、旧协议、旧配置、浏览器限制、脚本调用或第三方集成。
- 前端初始化、watch、默认回填、搜索条件同步、自动刷新是否会覆盖用户已经输入或选择的内容。
- 缓存列表、全局索引、反向索引、派生状态和本地状态是否在增删改后保持一致，并说明旧数据和缺失索引如何处理。
- 多端、多仓库、多语言 SDK 或跨服务场景是否只验证了主路径，遗漏调用方和边界场景。

### 6. 验证与可维护性

- 验证命令、静态检查、手工复测或未验证原因是否与任务风险匹配。
- 测试是否覆盖成功、失败、边界、兼容、并发、旧数据和回滚路径中最关键的风险。
- 是否引入重复实现、硬编码环境、魔法字符串、不可配置路径、临时开关或难以追踪的隐式依赖。
- 代码结构是否让下一个维护者能定位事实来源、失败原因和扩展点。

## 公共 API 设计视角

参考 `architect/references/checklist.md` 同名章节，重点关注：
- 新增字段是否向后兼容，删除或改名是否构成 Breaking Change
- 公共 API 是否稳定、可发现、可组合

## 领域扩展检查

- 涉及 Tunnel 且项目启用 Tunnel profile 时，遵守对应 `.agents/context/TUNNEL_RULES.md`，先说明影响范围；协议、Topic、gRPC、token、密钥变更只能输出审查意见，不能由审查角色直接修改。未启用 Tunnel profile 时，不要假设存在 `docs/tunnel-overview.md` 或 `docs/tunnel-protocol-alignment.md`。
- 涉及 IoT、MQTT、设备 SDK、Hook/gRPC、Swagger/proto、WebSocket、Redis、消息队列或浏览器限制时，按项目 profile 或任务说明补充对应链路检查；没有项目证据时，不把这些领域事实写成通用结论。
- 涉及前端体验时，额外检查用户输入保护、加载态、错误态、空态、响应式布局、可访问性和状态恢复。
- 涉及数据迁移、批处理、导入导出或审计留痕时，额外检查幂等、断点续跑、历史数据兼容、备份保留和可追溯性。
